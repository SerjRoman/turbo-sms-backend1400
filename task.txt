Ваша задача создать модуль Contact для Backend проекта TurboSMS. 
Модуль Контакт:

Модуль должен реализовывать функционал 3 эндпоинтов: 
    получение всех контактов, 
    получение информации о контакте по ID, 
    создание контакта. 

Все эндпоинты защищены authMiddleware.

Получение всех контактов - при отправке запроса клиент обязан добавить JWT токен и в ответ получить массив всех его(на основе ID из токена) контактов

Получение информации о контакте по его ID - клиент отправляет запрос в ссылке которого указывается динамический параметр - id. Если id не пришло, тогда нужно выдать ошибку BadRequestError. 
На основе этого id необходимо найти в БД сущность этого контакта и получить информацию о самом контакте и его связь с contactUser(информацию о юзере, который связан с контактом) 
    нужны только поля: id, username, lastSeenAt. Если такого контакта не существует - выдаем ошибку NotFoundError

Создание контакта - клиент отправляет запрос в формате FormData для создания контакта и с заголовком для JWT токен. 
Задача эндпоинта провалидировать данные из body: localName, avatar(если avatar null, тогда берется аватарка пользователя) и contactUserId // (придет строкой, поэтому нужно переделать в controller на number). 
Далее собрать все данные и создать сущность Contact, avatar - загруженная картинка размерами 300x300, ownerId - берется из JWT токена, contactUserId - id второго пользователя(с которым связываем сущность). 
В ответ на запрос отправляем созданную сущность Contact. В слое репозитория обрабатываем ошибку P2002 и выкидываем ConflictError(значит что такой контакт уже существует)

Реализовываете весь модуль по стандарту приложения( с типами из Prisma, контрактами, использованием мидделваеров, разделением на слои и тд)




Модуль User:
    Поиск User по username - клиент отправляет запрос с динамическим параметром username. Эндпоинт защищен authMiddleware. ()
    Эндпоинт должен найти пользователя в БД по username и вернуть его информацию(без пароля). Если пользователь не найден выдаем NotFoundError.
